# Auto Transfer Window Planner для Kerbal Space Program 1

Это исходники небольшого мода для KSP 1.x, который ищет оптимальное окно запуска между небесными телами в стиле Transfer Window Planner. Мод анализирует список `CelestialBody` из текущей игры, поэтому не привязан к stock-системе и может работать с RSS, KSRSS и другими planet pack.

## Что делает мод

- добавляет кнопку в стандартный тулбар KSP;
- даёт окно с выбором `Старт -> Цель`;
- сам ищет дату старта в заданном диапазоне лет;
- автоматически подбирает диапазон времени полёта вокруг гомановского перелёта;
- строит простую porkchop-карту `дата старта / время полёта / Δv`;
- считает:
  - дату старта;
  - дату прибытия;
  - время полёта;
  - суммарную Δv;
  - Δv вылета с круговой парковочной орбиты;
  - Δv захвата на круговую орбиту у цели;
  - `v∞` старта и прибытия;
  - `C3` старта;
  - фазовый угол;
  - угол и наклон вылета.

## Важное ограничение

Это не копия TriggerAu Transfer Window Planner и не полноценный навигационный комплекс. Расчёт сделан через patched conics и однооборотную задачу Ламберта: мод строит маршрут как последовательность двухтельных участков, а не как n-body симуляцию.

Поддерживаются все орбитальные тела, которые загружены в KSP:

- Kerbin -> Duna;
- Kerbin -> Mun;
- Mun -> Minmus;
- Earth -> Mars в RSS/KSRSS;
- Earth -> Moon;
- Moon -> Earth;
- Moon -> Mars.

Для прямых перелётов планета ↔ луна используется отдельная локальная оценка от/к круговой парковочной орбите. Точная миссия всё равно требует ручной донастройки maneuver nodes в игре.

## Как собрать DLL

Готовый DLL не приложен, потому что для сборки нужны DLL-файлы именно из твоей установленной KSP:

- `Assembly-CSharp.dll`;
- `Assembly-CSharp-firstpass.dll`;
- `UnityEngine*.dll`.

Они лежат в папке игры:

```text
Kerbal Space Program\KSP_x64_Data\Managed
```

### Способ 1: PowerShell

Открой PowerShell в папке проекта и выполни:

```powershell
.\Tools\build.ps1 -KspPath "C:\Program Files (x86)\Steam\steamapps\common\Kerbal Space Program"
```

Если KSP установлена в другом месте, замени путь.

После успешной сборки DLL появится здесь:

```text
GameData\AutoTransferWindowPlanner\Plugins\AutoTransferWindowPlanner.dll
```

### Способ 2: Visual Studio

1. Открой `Source\AutoTransferWindowPlanner.csproj`.
2. В свойствах проекта или через переменную `KSPDIR` укажи путь к KSP.
3. Собери проект в режиме `Release`.
4. Проверь, что DLL появилась в `GameData\AutoTransferWindowPlanner\Plugins`.

## Как установить в игру

После сборки скопируй папку:

```text
GameData\AutoTransferWindowPlanner
```

в папку:

```text
Kerbal Space Program\GameData
```

В игре открой Space Center, Tracking Station или Flight/Map View. В тулбаре появится кнопка планировщика.

## Как пользоваться

1. Выбери стартовое тело, например `Kerbin` или `Earth`.
2. Выбери цель, например `Duna`, `Mun`, `Mars` или `Moon`.
3. Оставь `Авто время полёта` включённым.
4. Укажи начальную и финальную орбиты, например 100 км.
5. Укажи, сколько лет искать, например 5.
6. Нажми `Искать окно`.

На карте:

- ось X — дата старта;
- ось Y — время полёта;
- цвет — суммарная Δv;
- белый крест — лучший найденный вариант.

## Формула оценки Δv

Для вылета с круговой парковочной орбиты используется приближение:

```text
Δv = sqrt(v∞² + 2μ/rp) - sqrt(μ/rp)
```

Та же формула используется для оценки торможения при захвате на круговую орбиту у цели.
